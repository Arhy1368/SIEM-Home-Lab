Цель: Создание домашней лаборатории для изучения процессов централизованного сбора, анализа и корреляции логов, а также для отработки навыков реагирования на инциденты.

📖 О проекте
Этот проект представляет собой полноценную лабораторную среду, в которой я развернул SIEM-систему на базе стека ELK (Elasticsearch, Logstash, Kibana). Основная цель — научиться обнаруживать и анализировать различные типы атак в реальном времени на изолированном полигоне. Лаборатория полностью воспроизводима и служит моим личным стендом для отработки навыков Blue Team.

🧩 Используемые технологии и инструменты
В проекте использовался следующий стек:

SIEM-платформа: Elastic Stack (Elasticsearch, Logstash, Kibana)

Агенты сбора логов: Winlogbeat (для Windows), Filebeat (для системных логов Linux и логов Suricata)

IDS/IPS: Suricata (для генерации сетевых событий)

Виртуализация: VirtualBox

Операционные системы: Ubuntu Server 22.04 LTS (для SIEM-сервера), Windows 11 Pro (жертва), Kali Linux (атакующий)

🎯 Практический анализ и сценарии угроз
Здесь я буду описывать конкретные сценарии атак, которые я проводил в лаборатории, с приложением скриншотов и описанием того, как SIEM помог их обнаружить.

Сценарий 1: Обнаружение брутфорс-атаки на RDP
Цель атаки: Попытка подбора пароля к RDP-сервису на Windows-клиенте.

Действия злоумышленника (Kali):
Использование утилиты hydra для перебора паролей:

bash
hydra -l administrator -P /usr/share/wordlists/rockyou.txt rdp://192.168.100.20
Что мы видим в SIEM (Kibana):

[будет скриншот дашборда Kibana с событиями 4625]

Пояснение: В логах Winlogbeat фиксируются множественные события Event ID 4625 (неудачный вход). Правило корреляции в Kibana настроено на подсчет количества таких событий с одного IP-адреса за короткий промежуток времени.

Результат:
Алерт сработал через 10 секунд, указав атакующий IP-адрес и целевую учетную запись.

Сценарий 2: Анализ подозрительной PowerShell-команды (атака "Живущий за счет земли")
Цель атаки: Попытка запустить вредоносную команду на Windows-клиенте, замаскированную под легитимную.

Действия злоумышленника:
Выполнение команды, которая использует Invoke-Expression для запуска скачанного скрипта.

Что мы видим в SIEM (Kibana):

[будет скриншот с найденным событием 4688 (создание процесса) или логами PowerShell]

Пояснение: Правило в Logstash выделяет подозрительные командные строки, содержащие шаблоны Invoke-, -EncodedCommand, DownloadString.

Результат:
Обнаружена попытка эксплуатации, создан алерт. Показывает важность мониторинга логов PowerShell.
